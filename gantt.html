Constructing Gantt Charts

Critical Path

  1. 1st step is to put in critical path
    1. Critical path:  the longest sequence of activities in a project that must be completed on time.
    2. Can be trivial (例： routine paper-work can be on a critical path), whilst expensive & imp. things exist that ≠ on critical path).
    3. Need critical path before creating ガント表
    4. Always drawn coming down in steps:
    5. 1 of reasons wherefore use steps is to show slippage:  what actually happened contra what should have happened
      1. Do not want there to be slippage on critical path ∵ anything going late on there means that every-thing else → late.
    6. However, 1st must discover critical path.
      1. Best method to obtain critical path is to use a network diagram
          1. Source:  Chris Croft Training
      2. Find critical path by adding up the times as you go across.
        1. Remember:  The critical path is the path with the longest times.
    7. If there is a task that can be overlapped with another on the critical path, then attempt to break the 1st one down a bit so that you can know when to begin the 2nd task.
      1. In so doing, you make the over-lap period as clear as possible
